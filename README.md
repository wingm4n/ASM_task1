# ASM_task1
Additional task #1, asm course. HSE SE 2026

Note: the program contains no malicious code, actually. 
It applies XOR cipher to payload section when executed first time;
after that, it unpacks the section at runtime and calls hidden functions.

# Light malware

Реализовать программу на языке ассемблера, выполняющую самораспаковку.

1. Реализовать функцию зашифрования с использованием [шифра гаммирования](https://ru.wikipedia.org/wiki/%D0%93%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5). Сигнатура функции на C выглядела бы так

`void xor(char *buf, size_t size, const char *key, size_t key_size)`

- `char *buf`       - изменяемый массив данных для выполнения гаммирования
- `size_t size`     - количество байт из массива `buf`
- `const char *key` - массив, содержащий гамму
- `size_t key_size` - количество байт гаммы

2. Реализовать функции в секции .text вычисления факториала числа `factor(n)` и нахождения n-ого числа фибоначчи `fibb(n)`. Проверить, отладить и убедиться, что работает.

3. Создать [секцию](https://www.nasm.us/doc/nasm09.html#section-9.10) `.payload` с правами на запись и выполнение. Перенести в нее код, реализованный в п.2

4. Реализовать функцию pack. Функция должна выполнять зашифрование гаммированием всей секции `.payload` на ключе, представляющем собой первые 512 байт из секции `.text`. Полученный шифр текст должен быть сохранен в тот же (или новый, если не получается) исполняемый elf файл по тем же адресам.

5. Реализовать функцию unpack. Функция должна на лету выполнить расшифрование секции `.payload`.


Note: для реализации функции pack необходимо в elf файле найти данные секции `.payload`, определить их размер и можно захардкодить это в программе. Сделать это можно, например в шестнадцатиричном редакторе, применив шаблон elf файла.

Первый запуск необходимо выполнить самостоятельно для зашифрования секции `.payload`.

При первом запуске нужно произвольным образом указать программе, что вы хотите выполнить упаковку. Упакованный elf нужно передать мне. Я выполню его запуск. Что должно произойти?! В результате работы программы, должна распаковаться секция `.payload` и выполниться функция вычисления факториала или n-ного числа фибоначчи. Программа должна корректно завершиться.
